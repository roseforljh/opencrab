# 项目计划目录说明

这个目录专门存放项目规划文件，目的是把“要做什么、先做什么、每个阶段做到什么程度算完成”固定下来，避免后续开发时方向漂移。

## 文件说明

- `master-plan.md`：总计划，记录项目目标、范围、技术方向、目录结构、阶段划分、交付物与验收标准。
- `phase-0-todo.md` 到 `phase-7-todo.md`：每个阶段的详细待办，后续执行时按阶段推进并逐项完成。

## 使用原则

1. 先看总计划，再进入具体阶段 TODO。
2. 每完成一个阶段，就回到这里更新对应文件状态。
3. 如果项目范围变化，先改计划文件，再改代码。
4. 所有说明统一使用中文，保持清晰、具体、可执行。
5. 当前执行策略已固定为：**前端优先完整实现，等用户确认满意后，再开始后端。**
