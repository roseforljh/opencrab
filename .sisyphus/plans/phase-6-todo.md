## 完成标准

1. 不改代码也能管理渠道、模型、路由、密钥和日志。
2. 基础限流和日志摘要可用。
3. 敏感信息不会直接泄露到日志。

## 验证清单

1. 通过管理 API 新增一个渠道，预期数据库内出现对应记录。
2. 通过管理 API 修改模型映射，预期无需改代码即可生效。
3. 不携带 API Key 请求代理接口，预期返回鉴权失败。
4. 连续高频请求触发限流，预期返回限流错误而不是服务崩溃。
5. 查询日志列表，预期能看到时间、模型、渠道、状态码、耗时等摘要字段。
