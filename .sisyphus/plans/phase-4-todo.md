# 阶段 4 TODO：后端基础骨架

## 阶段目标

等用户确认前端满意后，再把后端服务的基础启动能力搭起来，让服务能够稳定启动、加载配置、初始化数据库、输出日志。

## 待办清单

- [ ] 初始化 Go 项目。
- [ ] 创建 `cmd/api/main.go`。
- [ ] 建立 `internal/` 分层目录。
- [ ] 实现配置文件读取。
- [ ] 实现环境变量覆盖。
- [ ] 实现配置校验。
- [ ] 建立 SQLite 初始化逻辑。
- [ ] 建立 migration 机制。
- [ ] 建立结构化日志。
- [ ] 建立 request id 中间件。
- [ ] 实现 `/healthz`。
- [ ] 实现 `/readyz`。

## 完成标准

1. 服务可以成功启动。
2. 配置错误时能明确报错。
3. 数据库初始化成功。
4. 健康检查接口可用。

## 验证清单

1. 运行 `go run ./cmd/api`，确认服务可以启动。
2. 用 `curl http://127.0.0.1:<端口>/healthz`，预期返回健康状态。
3. 用 `curl http://127.0.0.1:<端口>/readyz`，预期返回就绪状态。
4. 删除或篡改必要配置后再次启动，预期服务直接报出明确配置错误。
5. 启动后检查 SQLite 文件或 migration 状态，确认数据库初始化完成。
