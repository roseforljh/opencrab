# 阶段 5 TODO：代理主链路

## 阶段目标

在用户确认前端满意后，再把最核心的代理链路打通，让系统真正能转发一次请求并返回结果。

## 待办清单

- [ ] 定义统一请求结构。
- [ ] 定义统一响应结构。
- [ ] 定义 provider 抽象接口。
- [ ] 接入第一个 provider。
- [ ] 实现 `/v1/chat/completions`。
- [ ] 支持普通响应。
- [ ] 支持 SSE 流式响应。
- [ ] 实现超时控制。
- [ ] 实现取消传播。
- [ ] 实现错误转换。
- [ ] 实现基础重试上限。

## 完成标准

1. curl 可成功请求。
2. 普通响应可返回。
3. 流式响应可返回。
4. 客户端中断后资源能正确释放。

## 验证清单

1. 使用 curl 调用 `/v1/chat/completions` 普通模式，请求体传入固定测试 prompt，预期返回合法 JSON 响应。
2. 使用 curl 调用 `/v1/chat/completions` 流式模式，预期返回连续 SSE 数据块。
3. 主动中断流式请求，检查服务日志，预期连接被正常关闭且无长时间悬挂请求。
4. 填入错误上游地址或错误密钥，预期系统返回统一错误结构，而不是原始崩溃信息。
5. 观察日志中的 request id、provider、耗时字段，确认代理链路基础观测已生效。
